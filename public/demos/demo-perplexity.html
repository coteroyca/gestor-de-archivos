Cómo leerlo

1. Primero ves el estado del negocio.
2. Luego entras a la dimensión más relevante.
3. El sistema muestra entidades afectadas.
4. Las señales explican el cambio de estado.
5. Las métricas cuantifican el impacto.

Casos por rol

1

Corporativo: primero sucursales, luego categorías, y después productos o clientes.

2

Operativo: primero abandonos, fricción, fill rate y causas probables.

3

Financiero: primero rentabilidad, luego mix de productos y segmentos de clientes.

La propuesta clave es usar estados operacionales como capa de entrada y dejar que cada rol navegue hacia el nivel que necesita: arriba hacia abajo, o desde el detalle hacia el patrón.

Arquitectura de navegación

Corporativo Holding → sucursales → categorías → productos → clientes.
Comercial Categorías → SKUs → RFM → cliente individual, si aplica.
Operativo Abandonos → checkout → fill rate → causa raíz.
Financiero Margen → rentabilidad → mix → segmentos.

Idea de negocio

Convierte el retail en un mercado cuantitativo: cada ticket es un tick, cada producto un activo, y cada cliente un estado que puede acelerarse, consolidarse o caer en riesgo.
